# ARCHITECTURE.md

# URL Shortener SaaS Platform
## Frontend Architecture

Version: 1.0

---

# Purpose

This document defines the architectural principles of the frontend.

Every feature generated by AI or implemented manually must follow this
architecture unless explicitly instructed otherwise.

The objective is to maintain:

- Scalability
- Maintainability
- Reusability
- Performance
- Clean separation of concerns

# API Architecture

Pages never communicate directly with the backend.

Correct Flow

Page

↓

Service

↓

Axios Instance

↓

Backend

Incorrect Flow

Page

↓

Axios.get()

↓

Backend

Never call Axios directly inside pages.

---

# Axios Instance

Create one centralized Axios instance.

Responsibilities

Base URL

Authorization Header

JWT Injection

Request Interceptors

Response Interceptors

401 Handling

Timeout

Error Processing

Every API call must use this instance.

# Architectural Rules

Always

✓ Reusable Components

✓ Service Layer

✓ Axios Instance

✓ Protected Routes

✓ Composition

✓ Tailwind Utilities

✓ React Query for Server Data

✓ Zustand for Global UI State

✓ React Hook Form

Never

✗ Duplicate Components

✗ Call Axios in Pages

✗ Hardcode API URLs

✗ Mix UI and Business Logic

✗ Create Huge Components

✗ Duplicate Layouts

✗ Ignore Loading States

✗ Ignore Error Handling

# Final Principle

The architecture should make the project easy to understand,
easy to extend, and easy to maintain.

Every generated feature should feel like it belongs to one cohesive
application rather than a collection of unrelated pages.

Consistency is more valuable than cleverness.
